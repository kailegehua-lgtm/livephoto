# LivePhoto 风格拍摄应用 PRD

## 1. 文档信息

- 产品名称：livephoto
- 版本：V0.4
- 状态：In Progress
- 日期：2026-05-24

## 2. 产品定位

本项目是一个 App 内自定义动态照片原型，不是系统标准 Live Photo，也不是产线检测工具。

当前目标：

1. 提供稳定的相机预览与拍摄体验。
2. 拍摄一张主图作为封面。
3. 生成与主图关联的短视频片段。
4. 在 App 内浏览、回看、删除、批量保存素材。
5. 以双模式验证“稳定闭环”和“前 3 秒回溯 + 后 1 秒补录”两条路径。

## 3. 产品目标

### 3.1 用户目标

1. 用户可以在 App 内拍摄动态照片。
2. 用户可以在列表中查看历史记录。
3. 用户可以长按详情页查看动态片段。
4. 用户可以对历史记录进行多选、删除、保存到系统相册。

### 3.2 工程目标

1. `iPhone 15 Pro` 上支持 `720p + 30fps` 预览。
2. 稳定模式保持流畅预览和可靠拍摄。
3. 实验模式支持接近前 3 秒、后 1 秒的动态片段导出。
4. 实验模式真实采样和编码链路在 `iPhone 15 Pro` 上接近 `30fps`。
4. 后台导出和文件落盘不能阻塞主线程。

## 4. 产品策略

### 4.1 稳定模式

目标是稳定、可回归、可展示：

1. 实时预览
2. 主图抓拍
3. 后 1 秒动态视频
4. 主图和视频可靠落盘
5. 列表和详情页可回看

当前稳定模式是默认兜底模式。

### 4.2 实验模式

目标是验证回溯抓拍价值：

1. 预缓存前 3 秒滚动编码片段
2. 点击拍摄后再补齐后 1 秒片段
3. 合成约 4 秒竖屏视频

当前实验模式采用滚动编码缓存方案，仍要求：

1. 预览可用
2. 状态可收敛
3. 导出完成后可播放
4. 相机回调与编码链路尽量接近 `30fps`

## 5. 非目标

当前版本不做：

1. 系统标准 Live Photo 资源生成
2. 系统相册中的原生 Live Photo 一体资产
3. 分享适配
4. 云同步
5. 编辑能力，如裁剪、滤镜、封面重选

## 6. 核心场景

### 6.1 拍摄

1. 用户进入拍摄页
2. 选择 `稳定模式` 或 `实验模式`
3. 看到实时预览
4. 点击拍摄
5. App 立即保存主图
6. 后台生成动态片段
7. 记录出现在列表中

### 6.2 浏览与回看

1. 用户进入 `我的动态照片`
2. 查看封面图、拍摄时间、模式标签、状态
3. 点击单条记录进入详情页
4. 在 `ready` 状态下长按播放动态片段

### 6.3 批量管理

1. 用户在列表页点击 `选择`
2. 选择单条或多条记录
3. 对选中记录执行：
   - 删除
   - 保存到系统相册

## 7. 功能范围

### 7.1 拍摄页

必须支持：

1. 相机实时预览
2. 模式切换：`稳定模式 / 实验模式`
3. 快门按钮
4. 实验模式说明
5. 当前 pre-roll 缓存秒数显示
6. 拍摄状态和导出状态提示
7. 错误提示

### 7.2 列表页

必须支持：

1. 展示历史记录
2. 显示封面图、拍摄时间、模式、状态
3. 普通浏览模式进入详情页
4. 选择模式下单选/多选
5. 批量删除
6. 批量保存到系统相册

### 7.3 详情页

必须支持：

1. 默认显示静态主图
2. `ready` 状态下长按播放动态片段
3. `processing / failed` 状态下显示降级提示

## 8. 状态设计

### 8.1 记录状态

每条 `MomentAsset` 必须处于以下单向状态之一：

1. `captured`
   主图已写入，动态片段尚未完成
2. `processing`
   后台正在导出动态片段
3. `ready`
   动态片段可播放
4. `failed`
   导出失败，但主图保留

要求：

1. 列表中必须能看到状态
2. 状态必须最终收敛到 `ready` 或 `failed`
3. 不允许记录长期停留在 `captured / processing`

### 8.2 页面状态

拍摄页关键状态：

1. `isCapturing`
2. `isBackgroundProcessing`
3. `processingPhase`
4. `availablePreRollSeconds`
5. `errorMessage`

要求：

1. 状态必须有清晰退出路径
2. 出错时不能让页面进入长期无反馈状态

## 9. 当前实现结论

截至当前版本，已有以下实现结果：

1. 稳定模式可稳定预览、拍摄、入列表、详情播放
2. 实验模式已切换到滚动编码缓存，不再使用原始像素帧深拷贝 pre-roll
3. 实验模式在 `iPhone 15 Pro` 上已验证 `cam≈30`、`buf≈30`
4. 实验模式缓存窗口当前约 `4.5s`，用于覆盖 `3s pre-roll + 1s post-roll` 和 segment 边界余量
5. 实验模式导出方向已修正，播放不再旋转 90 度
6. 列表页已支持选择、多选、删除、保存到系统相册

## 10. 保存到系统相册策略

当前“保存到系统相册”不是导出系统标准 Live Photo，而是：

1. 保存主图到系统相册
2. 对于 `ready` 记录，再额外保存视频到系统相册

因此在系统相册里的表现是：

1. 一张照片
2. 一个视频

而不是系统原生 Live Photo 一体资源。

## 11. 成功指标

### 11.1 稳定模式

1. 预览连续使用无明显卡顿
2. 拍照后记录稳定入列表
3. 动态片段可正常播放

### 11.2 实验模式

1. 预缓存秒数可超过 3 秒
2. 点击拍摄后可输出接近 4 秒的视频
3. 记录最终会收敛到 `ready` 或 `failed`
4. 视频方向正确
5. 真机调试中 `processingPhase` 能标识 `segments / empty / session / timeout` 等失败类型

### 11.3 管理能力

1. 多选删除稳定可用
2. 批量保存到系统相册稳定可用
